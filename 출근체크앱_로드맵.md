# 출근 체크 앱 - 개발 로드맵

## 추천 기술 스택

| 영역 | 기술 | 이유 |
|------|------|------|
| 프론트엔드 | **Next.js (React)** | 웹앱으로 설치 없이 PC/모바일 모두 접근 가능 |
| 백엔드/DB | **Firebase** | 실시간 DB, 소셜 로그인, 서버리스 함수 올인원 |
| 인증 | **Firebase Auth** | 카카오/구글 소셜 로그인 지원 |
| 실시간 동기화 | **Firestore 실시간 리스닝** | 출근 현황 즉시 반영 |
| 카카오톡 알림 | **카카오 메시지 API** | 출근 시 시카에게 카카오톡 발송 |
| 자동 초기화 | **Firebase Cloud Functions (Cron)** | 매일 자정 자동 실행 |
| 배포 | **Vercel + Firebase** | 무료 플랜으로 시작 가능 |

## 데이터 구조 (Firestore)

```
users/
  {userId}/
    name: "홍길동"
    role: "member" | "sika"   ← 시카는 "sika" 역할
    kakaoId: "..."

attendance/
  {날짜(YYYY-MM-DD)}/
    records/
      {userId}/
        name: "홍길동"
        location: "본사"
        checkedAt: 2026-03-29T09:15:00
        checked: true

locations/
  - "본사"
  - "현장A"
  - "현장B"
  (시카가 관리하는 목록)
```

## 개발 로드맵

### Phase 1 — 기획 & 환경 설정 (약 3일)

- [ ] Firebase 프로젝트 생성 및 Firestore, Auth 활성화
- [ ] Next.js 프로젝트 생성 (`npx create-next-app`)
- [ ] Firebase SDK 연결 설정
- [ ] 카카오 디벨로퍼스 앱 등록 (카카오 로그인 + 메시지 API 키 발급)
- [ ] Vercel 연동 (GitHub → 자동 배포 설정)

> 💡 이 단계가 제일 까다로울 수 있음. Firebase/카카오 콘솔 설정에 시간 투자 필요.

---

### Phase 2 — 소셜 로그인 구현 (약 3일)

- [ ] Firebase Auth로 **카카오 로그인** 연동 (커스텀 토큰 방식)
- [ ] Firebase Auth로 **구글 로그인** 연동 (상대적으로 쉬움)
- [ ] 최초 로그인 시 이름 설정 화면
- [ ] 시카 계정 역할(`role: "sika"`) 수동 지정

```
흐름:
카카오/구글 로그인 → Firestore에 유저 정보 저장 → 역할 확인 → 화면 분기
```

---

### Phase 3 — 일반 유저 출근 화면 (약 4일)

- [ ] 오늘 날짜 표시 및 이미 출근했는지 상태 확인
- [ ] 근무지 목록 드롭다운 (Firestore `locations` 컬렉션에서 불러오기)
- [ ] **출근 버튼** → Firestore `attendance/{오늘날짜}/records/{userId}` 저장
- [ ] 출근 완료 후 UI 변경 ("출근 완료 ✓" 표시, 버튼 비활성화)
- [ ] 당일 본인 출근 기록 표시 (몇 시에 어디로 출근했는지)

---

### Phase 4 — 시카 대시보드 (약 4일)

- [ ] `role === "sika"` 일 때만 접근 가능한 전용 페이지
- [ ] Firestore 실시간 리스닝으로 출근 현황 **즉시 업데이트**
- [ ] 오늘 날짜의 전체 멤버 출근 현황 표시
  - 출근 완료: 이름 + 근무지 + 시간 (초록색)
  - 미출근: 이름 (회색)
- [ ] 날짜 선택으로 과거 기록 조회 기능

```
화면 예시:
┌─────────────────────────────┐
│ 2026-03-29 출근 현황        │
├─────────────────────────────┤
│ ✅ 홍길동 — 본사 (09:15)   │
│ ✅ 이영희 — 현장A (08:50)  │
│ ⬜ 김철수 — 미출근          │
│ ⬜ 박민준 — 미출근          │
└─────────────────────────────┘
```

### Phase 5 — 카카오톡 알림 (약 3일)

- [ ] Firebase Cloud Functions에 출근 이벤트 트리거 함수 작성
- [ ] Firestore 문서 생성 감지 → 카카오 메시지 API 호출
- [ ] 시카의 카카오 계정으로 메시지 발송

```
알림 메시지 예시:
"[출근 알림] 홍길동님이 본사로 출근했습니다. (09:15)"
```

> ⚠️ 카카오 메시지 API는 친구 관계가 필요하거나, 비즈니스 채널 승인이 필요할 수 있음.
> 대안: 카카오톡 연동이 복잡하면 1단계는 이메일 알림으로 대체 가능.

### Phase 6 — 자동 초기화 (약 1일)

- [ ] Firebase Cloud Functions에 **매일 자정(00:00) Cron 함수** 설정
- [ ] 실제 데이터는 삭제하지 않고, 날짜별로 저장되어 있어 자동으로 "오늘" 기준으로 초기화됨
- [ ] (선택) 전날 미출근자 목록을 시카에게 요약 발송

```javascript
// Cloud Functions Cron 예시
exports.midnightReset = functions.pubsub
  .schedule('0 0 * * *')  // 매일 자정
  .timeZone('Asia/Seoul')
  .onRun(async (context) => {
    // 어제 날짜 미출근자 집계 & 알림 발송
  });
```

## 무료로 시작하는 방법

- **Firebase Spark 플랜 (무료)**: 소규모 사용에 충분
  - Firestore: 읽기 50,000회/일, 쓰기 20,000회/일
  - Cloud Functions: 월 125,000회 무료 호출
- **Vercel 무료 플랜**: 개인/소규모 프로젝트 배포 가능
- **카카오 디벨로퍼스**: 메시지 API 기본 무료

## 개발 시작 순서 (추천)

1. Firebase 프로젝트 + Next.js 세팅

3. 출근 버튼 → Firestore 저장 확인
4. 시카 대시보드 실시간 연동
5. 카카오 알림 추가
